<span style="font-size:20px;">**一、CI工程检查项概览**</span>

- 检查项分为静态检查、编译构建、测试3类，以下是当前CI工程检查项的详细情况：



| 检查项 |描述  |功能  |
|--|--|--|
| Build |编译构建  | 通过执行代码仓预设编译命令，完成源码编译、依赖包适配，对代码进行编译验证 | 
| anti_poison | 恶意代码检查	 | 恶意代码检查，支持检测文件中隐藏的病毒、以及恶意脚本（如恶意Shell/Python脚本等危险代码，保证代码安全| 
|  SCA| 开源片段检查 | 代码开源引用合规性检查，通过比对开源代码库，检测PR提交的代码片段是否存在开源代码复用/抄袭情况，若存在匹配项则触发告警，同时核查开源许可证兼容性，避免违规引用风险 | ||
|  codecheck| 编码规范检查 | 静态代码检查，核心覆盖两大维度：1. 安全类检查（算法逻辑安全性、敏感信息（密钥/明文密码等）清理情况、代码注入风险等）；2. 质量类检查（危险函数调用、重复文件/代码片段、代码规范违规、语法错误、潜在空指针/数组越界等问题），提前识别代码层面的潜在风险与缺陷 |
|  UT| 开发者测试 |单元测试任务，针对代码仓中的核心函数、类与模块，运行预设的单元测试（UT）代码，校验逻辑正确性。通过标准：测试用例运行无失败、代码覆盖率不低于代码仓预设阈值。为提升测试效率，部分大型代码仓会按模块拆分为多个UT子任务并行执行，缩短整体测试耗时  |

- 检查项任务状态说明
  
|  状态| 说明 | 备注 |
|--|--|--|
| running |任务执行中	  |  |
| waiting |任务待执行		  |  |
| FAILED |任务执行结果失败		  | 	任务执行失败，存在检查未通过、检查失败2种场景检查未通过：需要点击“日志”链接分析告警检查失败：需要通过日志查看失败原因 |

<span style="font-size:20px;">**二、问题定位指导**</span>

**<span style="font-size:20px;">PR 流水线异常排查操作指南</span>**

**1. 查看 PR 内容与流水线运行结果**

流水线运行结束后，系统会自动在 PR 评论区写入完整的运行结果。请重点关注任务状态，若存在异常任务项，其状态会明确标记为 “FAILED”。

点击流水线ID可查看整体流水线结果



**2. 跳转查看流水线日志**
![image.png](https://raw.gitcode.com/user-images/assets/9107584/00010513-b4ba-449a-b700-e0794f3e6bc3/image.png 'image.png')


**3. 分析日志中的错误信息**

查看详细日志发现检查项未通过
![image.png](https://raw.gitcode.com/user-images/assets/9107584/be823804-c38d-4f39-bbf1-37390aab0d09/image.png 'image.png')

**4. PR操作指导**

如上述查看日志发现检查项未通过，可以进行代码整改，完成后按以下操作指导拉起流水线

| 评论 |动作  |  
|--|--|
| rebuild |运行流水线  |  
| retry |重试流水线所有失败子任务  |  
| retry <任务名>  |仅重试指定失败子任务  |  
| stop |停止流水线  |
