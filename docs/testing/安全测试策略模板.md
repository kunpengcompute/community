# 产品安全测试策略模板
## 1、概述
该章节描述策略覆盖的范围，明确所覆盖的产品和测试活动。
参考：
- 网路安全测试的目标是通过安全测试，验证产品安全质量，发现系统中存在的安全脆弱性和风险，推动产品完成安全问题整改，从而提高产品安全性。本安全测试策略根据社区安全设计、安全编码要求、需求分析说明、特性设计说明等方面对产品网络安全测试覆盖内容进行分析，明确了测试范围、测试内容关注点、测试工具，用于指导网络安全测试方案和测试过程中计划制定，以对产品安全质量把关，提示产品安全质量。
## 2、产品研发现状分析
该章节对被测产品的研发状况进行分析描述，包括产品架构、安全特性、继承关系、使用场景、交付需求等。根据产品的研发状况确定正确的测试策略以达到最优的测试效果。
参考：
- xx产品新增xx、xx等特性，主要有以下变更：
  - 架构变更： 采用了全新的xx架构，解决了旧架构存在的安全问题。
  - 功能变更： 继承并增强xx功能，新增了xx、xx、xx等新特性。
  - 组网变更： 采用了4层架构，数据采集子系统->数据预处理子系统->数据存储子系统->数据展示子系统，全部采用xx服务器，数据采集子系统和数据预处理子系统部署在xx服务器上，数据存储子系统和数据展示子系统部署在xx服务器上。
